# Linux Admin Drill 4: Command Mastery Checklist

For each command below, write:

- what problem it helps with
- one signal you are looking for
- one common misuse

## Commands

- `systemctl status`
- `journalctl`
- `ps aux`
- `top`
- `pidstat`
- `free -m`
- `vmstat`
- `iostat`
- `df -h`
- `df -i`
- `findmnt`
- `lsblk`
- `ss -s`
- `ss -tanp`
- `ip addr`
- `ip route`
- `resolvectl status`
- `dig`
- `tcpdump`
- `iptables-save`
- `nft list ruleset`

## Mentor Tip

If you can explain what a command proves and what it does not prove, you are using it like an operator.
